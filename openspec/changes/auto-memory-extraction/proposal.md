## Why

当前 Jobopx Desktop 的长期记忆系统需要用户手动创建记忆，这导致很多有价值的对话内容（如用户偏好、项目决策、技术洞察）没有被保存。用户希望系统能够智能地自动识别并保存重要的对话内容，完全无需手动干预，让记忆系统真正"自动化"运行。

## What Changes

- 添加 AI 驱动的对话分析功能，自动识别值得记住的内容
- 实现完全自动记忆提取：AI 分析后直接保存，无需用户确认
- 在对话结束或达到一定轮次后自动触发分析和保存
- 集成去重检测，避免创建重复记忆
- 智能管理记忆数量，接近 200 个限制时自动清理旧记忆
- 添加用户偏好设置（启用/禁用自动提取、触发频率）
- 用户可以在记忆页面查看、编辑、删除自动创建的记忆

## Capabilities

### New Capabilities
- `memory-extraction`: AI 分析对话内容，识别关键信息并自动保存为记忆
- `extraction-triggers`: 配置何时触发自动记忆提取（对话结束、N 轮后、手动触发）
- `memory-auto-management`: 自动管理记忆数量，清理低价值或过期记忆

## Impact

- **后端**:
  - 新增 `src/memory/extractor.ts` - 核心提取逻辑
  - 新增 `src/memory/prompts.ts` - AI 分析提示词
  - 新增 `src/memory/auto-manager.ts` - 自动记忆管理
  - 修改 `src/conversation/manager.ts` - 集成提取触发器
  - 修改 `src/api/task-api.ts` - 添加手动触发 API 端点

- **前端**:
  - 在设置页面添加自动提取配置选项
  - 在记忆页面标记自动创建的记忆（可选）

- **性能**:
  - 额外的 AI 调用用于分析对话（异步非阻塞）
  - 建议使用轻量级模型（如 Haiku）降低成本
  - 维护对话游标，只分析新消息

- **用户体验**:
  - 完全自动化，零手动操作
  - 重要信息自动保存，不会丢失
  - 用户可以随时在记忆页面管理自动创建的记忆
  - 智能去重避免记忆冗余
